# Swaypik — дизайн-песочница

Это **изолированная копия фронтенда** Swaypik для работы над дизайном экранов.
Бэкенда и сервера здесь нет — все данные отдаёт мок (`src/api.js`), поэтому
приложение запускается одной командой и все экраны сразу рендерятся.

Главное: экраны здесь собраны на **тех же компонентах, токенах и стилях**, что и
боевой проект. Поэтому всё, что ты сделаешь тут, вставляется в основной проект
**copy-paste, без переписывания**.

---

## Запуск

Нужен установленный [Node.js](https://nodejs.org) версии 18+.

```bash
npm install
npm run dev
```

Откроется `http://127.0.0.1:5173`. Через ~2 сек после сплэша попадёшь в приложение.
Внизу — навигация по основным экранам (Поиск / Лайки / Мэтчи / Ачивки / Профиль).

> Это мобильное приложение (Telegram Mini App). Смотри в узком окне — открой
> DevTools (F12) → режим устройства (Ctrl+Shift+M) → ширина ~390px.

## Как посмотреть разные состояния экранов

Открой **`src/api.js`** — сверху блок `CONFIG`. Меняй флаги и сохраняй,
страница перезагрузится сама:

```js
const CONFIG = {
  scenario: 'app',     // 'app' | 'onboarding' | 'invite' | 'banned'
  premium: true,       // false — увидишь пейволлы и заблюренные лайки
  dailyReward: false,  // true — всплывёт модалка ежедневной награды
  emptyFeed: false,    // true — «люди рядом закончились»
  emptyLikes: false,   // true — пустой экран лайков
  emptyMatches: false, // true — пустой экран мэтчей
}
```

- `scenario: 'onboarding'` — пройти регистрацию (имя → пол → возраст → фото → гео → верификация)
- `scenario: 'invite'` — экран «вход по инвайту»
- `scenario: 'banned'` — экран бана
- Премиум-экран, чужие анкеты, мем-тест и модалки открываются кнопками внутри приложения.

## Где что лежит

```
src/
  screens/      ← экраны (это основное, что ты дорабатываешь)
  modals/       ← модалки / bottom sheets
  components/   ← общие компоненты
  components/ui/ ← дизайн-система: Screen, GlassCard, кнопки, бейджи и т.д.
  styles.css    ← все CSS-переменные (токены) и утилити-классы
  api.js        ← МОК (заглушка). НЕ редактировать кроме блока CONFIG.
```

**Прежде чем верстать — прочитай `CLAUDE.md`.** Там правила дизайн-системы:
какие токены, классы и готовые компоненты использовать. Если работаешь через
ИИ-агента (Claude Code / Codex и т.п.) — он подхватит `CLAUDE.md` автоматически.

## Чего НЕ трогать

Эти файлы — заглушки песочницы. Их **не нужно менять и не нужно присылать назад**:

- `src/api.js` — мок данных (в проекте он боевой)
- `src/components/Sticker.jsx` — эмодзи-заглушка (в проекте тянет анимации)
- `vite.config.js`

Если экрану нужны новые данные — не лезь в бэкенд, просто скажи об этом в
сопроводительном сообщении, подберём на нашей стороне.

---

## Как сдавать работу

Просто пришли изменённые файлы из `src/screens/`, `src/modals/`,
`src/components/` (по которым работал). Не нужно слать `node_modules`, `dist`
или весь проект целиком — только то, что менял.
